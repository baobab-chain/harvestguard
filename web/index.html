The mechanism

Two contracts: one watches the sky, one holds the money

  1. 01

    Weather providers report rainfall

    Multiple independent providers submit readings for a location. The oracle returns the median — no single source controls the outcome.

  2. 02

    Farmer buys a policy

    Pick a location, a rainfall threshold, and an expiry date. Pay a small premium into the shared pool.

  3. 03

    Anyone can check the trigger

    The condition is public and objective, so it doesn't need to be the farmer, the insurer, or an admin who calls it — anyone can fire a legitimate payout.

  4. 04

    Payout settles automatically

    If rainfall crossed the threshold, funds move from the pool to the farmer immediately. No investigation, no paperwork.

Why not Reflector, Stellar's existing oracle

Because rainfall isn't
an asset price

We checked. Reflector is excellent at what it does — asset prices, FX rates, exchange data — but it has no concept of rainfall or weather. HarvestGuard ships its own small, multi-provider weather oracle instead, deliberately built in a familiar, Reflector-like shape for anyone who's integrated with it before.
